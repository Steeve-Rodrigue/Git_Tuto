## Method 2: Pre-commit Framework (Hybrid)

The pre-commit framework manages hooks via a **version-controlled config file**. The hybrid approach combines **remote hooks** (community-maintained) and **local hooks** (your custom scripts).

---

## Why Pre-commit Framework?

| Feature | Local Hooks (Part 2) | Pre-commit Framework |
|---------|---------------------|---------------------|
| Version controlled | ❌ No | ✅ Yes |
| Team sharing | ❌ Manual | ✅ Automatic |
| Updates | ❌ Manual | ✅ `pre-commit autoupdate` |
| Ready-to-use hooks | ❌ Build yourself | ✅ Thousands available |

**Hybrid = Best of both worlds:**
- Remote hooks for standard tools (Ruff, ESLint)
- Local hooks for custom project rules

## Project Structure
```
my-project/
├── .git/
│   └── hooks/
│       └── pre-commit              # Auto-generated by pre-commit
├── .pre-commit-config.yaml         # Hook configuration (tracked)
├── pyproject.toml                  # Ruff config (from Part 1)
├── hooks/                          # Custom local hooks (tracked)
│   ├── ruff_custom.py
│   └── custom_checks.py
├── src/
│   ├── __init__.py
│   └── main.py
└── README.md
```

**Key difference from Part 2:**
- `.pre-commit-config.yaml` → Version controlled ✅
- `hooks/` directory → Version controlled ✅
- `.git/hooks/pre-commit` → Auto-generated, don't edit

## Hybrid Configuration

### Basic Setup

Create `.pre-commit-config.yaml` in repository root:
```yaml
# .pre-commit-config.yaml

repos:
  # -------------------------------------
  # REMOTE HOOKS: Community maintained
  # -------------------------------------
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.14.14
    hooks:
      - id: ruff-format      # Auto-format
      - id: ruff-check       # Lint and check

  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: detect-private-key

  # -------------------------------------
  # LOCAL HOOKS: Your custom scripts
  # -------------------------------------
  - repo: local
    hooks:
      - id: ruff-custom
        name: Ruff Custom Messages
        entry: python hooks/ruff_custom.py
        language: system
        types: [python]

      - id: custom-checks
        name: Project Checks
        entry: python hooks/custom_checks.py
        language: system
        types: [python]
        pass_filenames: true
```

**Configuration explained:**
- **Remote hooks**: Use `repo: https://...` with version `rev:`
- **Local hooks**: Use `repo: local` pointing to your scripts in `hooks/`
- **Types**: Filter by file type (`[python]`, `[javascript]`, etc.)
- **pass_filenames**: Pass file paths as arguments to script

## Custom Local Hooks

### Create hooks/ Directory
```bash
mkdir -p hooks
```

### Hook 1: Custom Ruff Messages
```python
#!/usr/bin/env python3
# hooks/ruff_custom.py

import subprocess
import sys

print("=== Custom Ruff Hook ===")

# Get staged Python files
result = subprocess.run(
    ["git", "diff", "--cached", "--name-only", "--diff-filter=d"],
    capture_output=True, text=True
)
files = [f for f in result.stdout.splitlines() if f.endswith(".py")]

if not files:
    print("No Python files")
    sys.exit(0)

print(f"Checking: {', '.join(files)}")

# Run Ruff
check = subprocess.run(["ruff", "check"] + files)

if check.returncode != 0:
    print("\n⚠ Commit blocked: Fix Ruff errors")
    sys.exit(1)

print("✅ Passed")
sys.exit(0)
```

### Hook 2: Custom Checks
```python
#!/usr/bin/env python3
# hooks/custom_checks.py

import sys

print("=== Custom Checks ===")

files = sys.argv[1:]
errors = False

for f in files:
    content = open(f).read()
    
    if 'print(' in content:
        print(f"❌ {f}: Found print()")
        errors = True
    
    if 'except:' in content:
        print(f"❌ {f}: Found bare except")
        errors = True

if errors:
    sys.exit(1)

print("✅ OK")
sys.exit(0)
```

### Make Executable
```bash
chmod +x hooks/ruff_custom.py
chmod +x hooks/custom_checks.py
```

## Usage Workflow

### First Time Setup
```bash
# 1. Install pre-commit
pip install pre-commit

# 2. Create config file
touch .pre-commit-config.yaml
# (add configuration from above)

# 3. Create hooks directory
mkdir hooks
# (add custom scripts)

# 4. Install hooks
pre-commit install

# 5. Run on all files
pre-commit run --all-files
```

### Daily Usage
```bash
# Hooks run automatically
git add .
git commit -m "feat: add feature"

# Output:
# Ruff format.....................Passed
# Ruff check......................Passed
# Ruff Custom Messages............Passed
# Project Checks..................Passed
# [main abc1234] feat: add feature
```

### Team Member Setup
```bash
# 1. Clone repository
git clone <repo-url>
cd <repo>

# 2. Install pre-commit
pip install pre-commit

# 3. Install hooks (reads .pre-commit-config.yaml)
pre-commit install

# Done! Hooks ready to use
```

## Commands

### Essential Commands
```bash
# Install hooks
pre-commit install

# Run manually on all files
pre-commit run --all-files

# Run specific hook
pre-commit run ruff-format --all-files

# Update to latest versions
pre-commit autoupdate

# Bypass hooks (emergency only)
git commit --no-verify -m "message"
```

### Hook Configuration Options
```yaml
- repo: local
  hooks:
    - id: my-hook
      name: Display Name
      entry: python hooks/script.py
      language: system
      types: [python]              # File types
      files: \.py$                 # Regex pattern
      exclude: ^tests/             # Exclude pattern
      pass_filenames: true         # Pass files as args
      stages: [commit]             # When to run
```

### File Type Filters
```yaml
types: [python]              # .py files
types: [javascript]          # .js files
types: [yaml]                # .yaml/.yml files
types_or: [python, pyi]      # Multiple types
```

## Comparison

| Aspect | Part 2 (Local) | Part 3 (Pre-commit) |
|--------|---------------|---------------------|
| **Location** | `.git/hooks/` | `.pre-commit-config.yaml` + `hooks/` |
| **Version control** | ❌ No | ✅ Yes |
| **Team setup** | Manual copy | `pre-commit install` |
| **Updates** | Manual | `pre-commit autoupdate` |
| **Remote hooks** | ❌ No | ✅ Yes (Ruff, ESLint, etc.) |
| **Custom hooks** | ✅ Yes | ✅ Yes (in `hooks/`) |
| **Best for** | Personal | Teams |

## Quick Reference

### Update Hooks
```bash
# Update all to latest
pre-commit autoupdate

# Commit changes
git add .pre-commit-config.yaml
git commit -m "chore: update hooks"
```

### Troubleshooting
```bash
# Validate config
pre-commit validate-config

# Run verbose
pre-commit run --all-files --verbose

# Clear cache
pre-commit clean
```

### Skip Specific Hook
```bash
# Skip one hook
SKIP=ruff-format git commit -m "message"

# Skip all
git commit --no-verify -m "message"
```
